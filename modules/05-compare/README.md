# โมดูล 5 — เปรียบเทียบ 3 ฐานข้อมูล: ใช้อันไหนเมื่อไหร่?

## 🎯 เป้าหมาย
เข้าใจว่าไม่มีฐานข้อมูล "ดีที่สุด" — มีแต่ "เหมาะกับงานไหน"

## 📊 ตารางเปรียบเทียบ

| | 🐘 PostgreSQL | 🍃 MongoDB | 🔎 ElasticSearch |
|---|---|---|---|
| **ประเภท** | Relational | Document | Search engine |
| **โครงสร้างข้อมูล** | ตาราง (ตายตัว) | เอกสาร JSON (ยืดหยุ่น) | เอกสารที่ทำ index เพื่อค้น |
| **เก่งเรื่อง** | ความถูกต้อง, JOIN, transaction | ข้อมูลโครงสร้างไม่ตายตัว | ค้นหา full-text เร็ว |
| **ไม่เหมาะ** | ข้อมูลโครงสร้างเปลี่ยนบ่อย | งานที่ต้อง JOIN ซับซ้อน | เป็นแหล่งข้อมูลหลัก (source of truth) |
| **ตัวอย่างในร้านเรา** | ออเดอร์, สต็อก, เงิน | รีวิว, สเปคสินค้า | ช่องค้นหาสินค้า |

## 💡 หลักคิดง่ายๆ
- ต้องการ **ความถูกต้องของเงิน/สต็อก** → PostgreSQL
- ข้อมูล **รูปแบบหลากหลาย เปลี่ยนบ่อย** → MongoDB
- ต้องการ **ช่องค้นหาที่ฉลาดและเร็ว** → ElasticSearch

ในระบบจริง มักใช้ **หลายตัวร่วมกัน** (เหมือน workshop นี้!) — PostgreSQL เก็บข้อมูลหลัก
แล้ว sync ไป ElasticSearch เพื่อทำหน้าค้นหา ส่วน MongoDB เก็บข้อมูลที่ยืดหยุ่น
