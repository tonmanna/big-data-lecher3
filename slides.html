โมดูล 1🐳 Docker — ยก stack ด้วยคำสั่งเดียว

Container = "กล่องสำเร็จรูป" ที่มีโปรแกรม + ทุกอย่างพร้อมรัน ไม่ต้องลงทีละตัว

cp .env.example .env
docker compose up -d      # เปิดทุก service
docker compose ps         # เช็คสถานะ

เปิดหน้าเว็บได้เลย: Adminer :8080 • Mongo Express :8081 • Kibana :5601

🤖 ถาม Claude: "elasticsearch ขึ้น unhealthy ช่วยดู log แล้วบอกสาเหตุ"

โมดูล 2🐘 PostgreSQL — ตารางและ JOIN

ข้อมูลเก็บเป็นตารางที่เชื่อมกันด้วยความสัมพันธ์ (relation)

-- ยอดขายรวมของลูกค้าแต่ละคน
SELECT c.full_name, SUM(oi.quantity * oi.unit_price) AS total
FROM customers c
JOIN orders o       ON c.customer_id = o.customer_id
JOIN order_items oi ON o.order_id = oi.order_id
GROUP BY c.full_name
ORDER BY total DESC;
🤖 ถาม Claude: "หาลูกค้าที่ใช้เงินมากสุด 3 คน ไม่นับออเดอร์ที่ cancelled"

โมดูล 3🍃 MongoDB — ข้อมูลแบบเอกสาร

แต่ละ document มี field ไม่เท่ากันได้ และเก็บข้อมูลซ้อนกัน (nested) ได้

// คะแนนเฉลี่ยและจำนวนรีวิวต่อสินค้า
db.reviews.aggregate([
  { $group: {
      _id: "$product_name",
      avg_rating:  { $avg: "$rating" },
      review_count:{ $sum: 1 }
  }},
  { $sort: { avg_rating: -1 } }
])
🤖 ถาม Claude: "นับว่าแต่ละ tag ในรีวิวถูกใช้กี่ครั้ง"

โมดูล 4🔎 ElasticSearch — ค้นหาแบบฉลาด

ช่องค้นหาที่ SQL ทำได้ไม่ดี — เร็ว + จัดอันดับความเกี่ยวข้องให้

GET products/_search
{
  "query": {
    "bool": {
      "must":   { "match": { "name": "keyboard" } },
      "filter": [{ "range": { "price": { "lte": 3000 } } }]
    }
  }
}
🤖 ถาม Claude: "หาหนังสือที่มีคำว่า data ในชื่อหรือคำอธิบาย"

โมดูล 5⚖️ ใช้ฐานข้อมูลไหนเมื่อไหร่?

🐘 Postgres🍃 Mongo🔎 Elastic
เก่งเรื่องความถูกต้อง JOINโครงสร้างยืดหยุ่นค้นหาเร็ว
ในร้านเราออเดอร์ เงิน สต็อกรีวิว สเปคช่องค้นหา

ไม่มี "ดีที่สุด" มีแต่ "เหมาะกับงานไหน" — ระบบจริงมักใช้หลายตัวร่วมกัน

โมดูล 6🔄 Airbyte — ท่อส่งข้อมูลอัตโนมัติ

ตั้งครั้งเดียว มันดึงข้อมูลจาก Source → Destination ให้ตามรอบ

curl -LsfS https://get.airbyte.com | bash -
abctl local install        # เปิด Airbyte
abctl local credentials    # เอารหัสเข้า http://localhost:8000

Source = PostgreSQL ของเรา → Destination เริ่มที่ Local JSON (ง่ายสุด)

🤖 ถาม Claude: "ต่อ Source Postgres ไม่ติด ขึ้น connection refused เช็คอะไรบ้าง"

โมดูล 7🔗 ต่อทุกอย่างเป็น Pipeline เดียว

ลูกค้าสั่งซื้อ → บันทึกที่ PostgreSQL
↓
Airbyte ดึง products ไป
↓
ElasticSearch ทำ index ค้นหา
↓
ผู้ใช้ค้นหาสินค้าบนหน้าเว็บได้ทันที ✨

Postgres = แหล่งความจริง • Airbyte = สายพาน • Elastic = ชั้นค้นหา

โมดูล 8🎓 โปรเจกต์จบ

รวมทุกอย่างที่เรียนมา สร้างระบบข้อมูลร้านของคุณเอง (เลือกทำ 3 ข้อ)

  • รายงานยอดขายแยกตามหมวด (SQL)
  • หา top reviewer (MongoDB aggregation)
  • หน้าค้นหาสินค้า + กรองราคา (ElasticSearch)
  • sync ข้อมูลด้วย Airbyte ไป destination
  • ตรวจ data quality / เพิ่ม dataset ของตัวเอง
🤖 ใช้ Claude Code เป็นคู่หู: ออกแบบ → เขียนโค้ด → debug → อธิบาย
